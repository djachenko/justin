# Метафайлы (_meta.json)

## Концепция

Каждая папка может иметь `_meta.json` рядом с собой. Метафайлы хранят ID поста VK, ID группы, статус публикации и другие данные, привязанные к конкретной папке.

Контекст накапливается **сверху вниз** по дереву папок: данные из родительской папки доступны в дочерней.

## Типы метафайлов

| Тип | Что хранит |
|---|---|
| `RootMetafile` | Базовый тип, основа для остальных |
| `PhotosetMetafile` | Состояние фотосета, ID публикации |
| `PostMetafile` | ID поста VK, статус |
| `PersonMetafile` | Данные о человеке |
| `CommentMetafile` | Комментарий VK |
| `AlbumMetafile` | ID альбома VK |
| `DriveMetafile` | Данные диска |
| `LocationMetafile` | Геолокация |

## Расположение кода

`shared/metafiles/` — чтение, запись, миграции метафайлов.

## Миграции метафайлов

При изменении формата данных — миграция через `photoset_migration.py` в `shared/models/`. Паттерн аналогичен общему паттерну миграций: ключ (или `None` = каждый запуск), выполняется один раз.

## Чтение/запись

Метафайлы читаются лениво. При записи — перезаписывается весь файл. Fail-fast: если директория недоступна — ошибка при первом обращении.
